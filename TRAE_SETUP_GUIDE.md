# GitHub GUI 启动器技能 - Trae 设置指南

本文档详细说明如何将 GitHub GUI 启动器技能添加到 Trae 设置中。

## 前提条件

- 已安装 Trae IDE
- 已下载 GitHub GUI 启动器技能文件
- 已安装 GitHub Desktop 客户端

## 技能文件结构

确保您的技能目录结构如下：

```
github_gui_launcher/
├── README_trae.md      # Trae 平台专用说明
├── config.json         # 技能内部配置文件
├── config_input.json   # Trae 输入参数配置
├── requirements.txt    # Python 依赖声明
├── skill.py            # 主执行逻辑
├── skill.yaml          # 技能元数据
└── TRAE_SETUP_GUIDE.md # 本设置指南
```

## 步骤 1：添加技能到 Trae

1. **打开 Trae IDE**：启动 Trae 开发环境

2. **进入技能管理页面**：
   - 点击左侧导航栏中的「技能」选项
   - 或使用快捷键 `Ctrl+Shift+S`

3. **添加本地技能**：
   - 点击「添加技能」按钮
   - 选择「本地技能」选项
   - 浏览并选择您的 `github_gui_launcher` 目录
   - 点击「确定」按钮

4. **配置技能**：
   - 检查技能信息是否正确
   - 确认依赖项是否已安装
   - 点击「保存」按钮

## 步骤 2：创建技能触发规则

1. **进入规则管理页面**：
   - 点击左侧导航栏中的「规则」选项
   - 或使用快捷键 `Ctrl+Shift+R`

2. **创建新规则**：
   - 点击「新建规则」按钮
   - 输入规则名称，如「GitHub 仓库启动器」

3. **配置触发条件**：
   - 选择「关键词触发」
   - 添加触发关键词，如：
     - "github"
     - "git"
     - "启动器"
     - "打开仓库"

4. **配置动作**：
   - 选择「执行技能」
   - 从下拉菜单中选择「GitHub GUI 启动器」
   - 配置参数映射：
     - 将用户输入映射到 `github_path` 参数
     - 设置 `client_type` 参数为 "default"

5. **保存规则**：
   - 点击「保存规则」按钮
   - 启用规则

## 步骤 3：测试技能

1. **在 Trae 聊天窗口中测试**：
   - 输入：`github_gui_launcher技能，打开：https://github.com/sansan0/TrendRadar`
   - 检查技能是否成功启动 GitHub Desktop 并打开指定仓库

2. **测试本地路径**：
   - 输入：`github_gui_launcher技能，本地部署：D:\Projects\TrendRadar`
   - 检查技能是否成功识别并打开本地仓库

3. **测试无参数启动**：
   - 输入：`github_gui_launcher技能`
   - 检查技能是否成功启动 GitHub Desktop

## 步骤 4：技能管理和更新

### 更新技能

1. **修改技能文件**：
   - 对技能文件进行必要的修改

2. **刷新技能**：
   - 在 Trae 技能管理页面找到 GitHub GUI 启动器技能
   - 点击「刷新」按钮
   - 确认更新是否成功

### 禁用/启用技能

1. **在技能管理页面**：
   - 找到 GitHub GUI 启动器技能
   - 点击「禁用」按钮暂时禁用技能
   - 点击「启用」按钮重新启用技能

### 删除技能

1. **在技能管理页面**：
   - 找到 GitHub GUI 启动器技能
   - 点击「删除」按钮
   - 确认删除操作

## 常见问题及解决方案

### 问题 1：技能无法启动 GitHub Desktop

**解决方案**：
- 检查是否已安装 GitHub Desktop
- 检查 `config.json` 文件中的客户端路径是否正确
- 尝试使用管理员权限运行 Trae IDE

### 问题 2：技能无法识别本地仓库路径

**解决方案**：
- 检查本地仓库路径是否正确
- 确保路径中不包含特殊字符
- 尝试使用绝对路径

### 问题 3：技能在 Trae 中显示为未启用

**解决方案**：
- 在技能管理页面检查技能状态
- 确保所有依赖项都已安装
- 检查技能文件是否完整
